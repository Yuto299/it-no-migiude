記事SEOと回遊を強化するためのタスクリスト。優先度は上から順。

## 1. GA4 を動かす（最優先）

現状、`NEXT_PUBLIC_GA_ID` が `.env.local` 未設定で、計測タグが本番でも動いていない可能性が高い。アクセスデータがないと「人気順」「リライト対象選定」「効果測定」のいずれも進められないため最優先。

### やること

- [ ] GA4 管理画面で測定ID（`G-XXXXXXXXXX`）を取得
  - 管理 → データストリーム → ウェブストリーム
- [ ] Vercel の Environment Variables に `NEXT_PUBLIC_GA_ID` を追加
- [ ] ローカル `.env.local` にも追加
- [ ] 本番デプロイ後、GA4 リアルタイムレポートでヒットを確認
- [ ] 1〜2週間データをためる

### コード側

`src/app/layout.tsx:85-100` に Script タグは実装済み。環境変数を入れるだけで動く。

## 2. GA4 Data API を叩けるようにする（人気順の前提）

「人気順」並び替えや「アクセス上位記事のリライト」を実装するためには Data API が必要。

### やること

- [ ] Google Cloud Console でサービスアカウントを作成
  - 既存プロジェクトでも新規でもOK
- [ ] サービスアカウントの JSON 鍵をダウンロード
- [ ] GA4 管理画面 → プロパティのアクセス管理 で、そのサービスアカウントのメールに **閲覧者** 権限を付与
- [ ] GA4 プロパティID（管理画面左下の数字）をコピー
- [ ] `.env.local` と Vercel に追加:
  - `GA4_PROPERTY_ID=<数字>`
  - `GOOGLE_APPLICATION_CREDENTIALS_JSON=<JSON鍵の中身を1行JSONに>` または鍵をBase64で保存
- [ ] `npm install @google-analytics/data`
- [ ] `src/lib/ga4.ts` を作成（ページビュー上位を取得する関数）

## 4. 並び替え機能（新着順 / 人気順 / おすすめ順）

記事一覧・カテゴリ一覧で並び替えタブを実装する。

### やること

#### 4-1. microCMS スキーマ更新
- [ ] `articles` に `featured`（ブール型）フィールドを追加
- [ ] 既存記事をいくつか `featured: true` に設定

#### 4-2. 一覧ページに並び替えUI
- [ ] `src/components/article/SortTabs.tsx` を作成（タブUI）
- [ ] クエリパラメータ `?sort=latest|popular|featured` で切り替え
- [ ] 一覧ページ（`/`、`/articles`、`/categories/[slug]`）に組み込み

#### 4-3. 各並び順のロジック
- [ ] **新着順**: `orders=-publishedAt`（実装済み・デフォルト）
- [ ] **人気順**: GA4 Data API のページビュー上位 → microCMS 記事と紐付け
  - 1日1回バッチで取得 → Vercel KV or ファイルキャッシュに保存
  - GA4 Data API 設定（タスク2）が前提
- [ ] **おすすめ順**: `featured: true` を上位、同点は新着順
  - 必要になったら `featured` を `recommendScore`（数値）に拡張

#### 4-4. 人気順の実装が間に合わない場合
- [ ] 「人気順」タブは GA4 Data API 設定が済むまで非表示にする
