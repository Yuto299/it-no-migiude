# ITの右腕 — サイト設計・ワイヤーフレーム仕様書

Ver. 1.0 | 2025年3月

---

## 1. プロダクト概要

「ITの右腕」は、中小企業・スタートアップ向けのDX・IT活用メディアです。記事を通じて信頼を積み上げ、問い合わせ・無料相談から案件獲得につなげるメディア×セールスの複合プロダクトです。

### 1-1. ミッション

現場で使えるDX・IT活用のヒントを、難しい言葉なしに届ける。

### 1-2. ターゲットユーザー

- 中小企業の経営者・役員（決裁者）
- 現場の業務担当者（課題を持つ人）
- IT担当・情シス（比較検討者）
- スタートアップ（スピード重視）

### 1-3. 案件獲得フロー

| ステップ | 内容 |
|---|---|
| ① 記事読了 | 検索・SNS経由でターゲットが記事に到達 |
| ② CTA | 記事内バナー or フッターCTAで誘導 |
| ③ フォーム | /contact または /consultation へ遷移 |
| ④ 通知 | Resendでメール通知 → 運営者に即時届く |
| ⑤ 商談 | 30分無料相談 → クロージング → 受注 |

## 4. 機能要件

### 4-1. P0（必須）

- 記事一覧・詳細ページ（SSG）
- カテゴリ・タグ絞り込み
- 問い合わせフォーム（名前 / 会社名 / メールアドレス / お問い合わせ内容）
- 無料相談30分 予約フォーム（Calendly埋め込み）
- OGP・メタタグ（SEO基盤）
- レスポンシブ対応（モバイルファースト）

### 4-2. P1（次フェーズ）

- 記事内CTA（本文中バナー埋め込み）
- 関連記事表示（記事詳細下部）
- パンくずリスト
- サイトマップ自動生成（next-sitemap）

### 4-3. P2（将来）

- メルマガ登録機能
- 事例・実績ページ
- サービス紹介ページ

## 6. ワイヤーフレーム仕様

### 6-1. トップページ（/）

7セクション構成でメディアの価値訴求と案件獲得導線を設計。

| # | セクション | 役割・内容 |
|---|---|---|
| ① | ナビゲーション | ロゴ / 記事一覧 / カテゴリ / 運営者紹介 / 無料相談ボタン / お問い合わせボタン |
| ② | ヒーロー | キャッチコピー・サブコピー・CTA2つ（無料相談 / 記事を読む）。ブルー背景で訴求強度を上げる |
| ③ | 新着記事 | 3カラムカード表示。サムネイル・カテゴリバッジ・タイトル・日付。「一覧を見る」リンク |
| ④ | カテゴリ一覧 | 3カラム。アイコン・カテゴリ名・記事数。クリックで /categories/[slug] へ |
| ⑤ | 中間CTA | グリーン背景で視線を強制誘導。「無料30分相談 / オンライン / 完全無料」で心理的ハードルを下げる |
| ⑥ | 運営者紹介 | アイコン・名前・肩書き・スキルタグ（FastAPI / Next.js / SAP / DX支援）で信頼性担保 |
| ⑦ | フッター | 3カラム（サイト説明 / コンテンツリンク / お問い合わせリンク）+ 特商法・プライバシーポリシー |

### 6-2. 記事詳細ページ（/articles/[slug]）— 次フェーズ

記事本文の中間・末尾にCTAバナーを配置し、無料相談・問い合わせへの導線を確保。

- ページ上部: パンくずリスト + タイトル + サムネイル + 公開日 + カテゴリ
- 本文中: 中間CTA（記事の半分程度に埋め込み）
- 本文下: 関連記事3件 + 末尾CTA

### 6-3. 問い合わせ・相談フォーム（/contact, /consultation）— 次フェーズ

- `/contact`: React Hook Form + Resend。バリデーション付き。送信後サンクスページへ
- `/consultation`: Calendlyをiframe埋め込み。カレンダー表示で即予約完結

## 7. 非機能要件

| 項目 | 目標値・方針 |
|---|---|
| Lighthouseスコア | 90以上（Performance / SEO / Accessibility） |
| 画像最適化 | next/image使用。WebP変換・遅延ロード |
| ISR | 記事更新を最短60秒で反映 |
| OGP | 全ページにog:title / og:description / og:imageを設定 |
| サイトマップ | next-sitemapで自動生成 / Google Search Console登録 |
| アクセシビリティ | alt属性必須・フォームのラベル対応・キーボード操作 |

## 8. 開発優先順位

| フェーズ | タスク | ゴール |
|---|---|---|
| Phase 1 | CMS連携 + 記事表示（一覧・詳細） | コンテンツが動く状態 |
| Phase 2 | トップページ実装（ワイヤー通り） | ランディングできる状態 |
| Phase 3 | 問い合わせ・相談フォーム | リード獲得できる状態 |
| Phase 4 | SEO設定（OGP・サイトマップ・metaタグ） | 検索流入の土台 |
| Phase 5 | CTA設計・デザイン磨き・GA4設定 | 計測・改善できる状態 |

> Phase 1〜3を最速で完了させ、実際の問い合わせを受けながらPhase 4〜5を進める。
