# 記事作成フロー

ITの右腕メディアの記事を1本作成し、microCMSへ下書き投稿するまでを自動化するワークフロー。
毎日1本ペースで運用するための定型手順。

---

## 前提として読む（必要な範囲だけ）

- `docs/09_article_automation.md` — フォーマット仕様（H2+H3、`<hr>`区切り、CTA締め、文字数2,000〜3,500字、tags・metaDescription仕様）
- `knowledge/article-ideas.md` — 投稿候補のネタリスト（`[ ]` が未投稿）

CLAUDE.md と他のdocs/* は記事作成では基本不要なので読まなくてよい。

---

## 実行ステップ

### 1. ネタ候補の提示

1. `knowledge/article-ideas.md` を読み、未投稿（`[ ]`）のネタを把握する
2. microCMS の `articles` を `mcp__microcms__microcms_get_list`（fields=`id,title,slug,category,tags,publishedAt`、orders=`-publishedAt`、limit=10）で取得し、直近の傾向と被りを確認する
3. **未投稿リストの中から** 3〜4件を AskUserQuestion で提示してユーザーに選ばせる
   - カテゴリのバランス（AI活用 / DX入門 / 業務効率化）を意識
   - 直近数本と同じカテゴリばかり続けない
   - 各選択肢に「なぜこのトピックが今オススメか」を短く添える

### 2. 記事生成

選ばれたトピックで記事HTMLを生成する。**フォーマットは厳守**：

- 文字数: 2,000〜3,500字（日本語本文のみカウント）
- 見出し: H2（大テーマ）＋ H3（細分化）の2段構造のみ。H1・H4以下は使わない
- セクション区切り: 各 H2 の前に `<hr>` を挿入
- 冒頭: 見出しなしで読者の課題提起を 2〜3 段落
- 締め: 必ず「まとめ：〇〇」H2 セクションを置き、最後の段落で「30分の無料相談でご状況を聞かせてください」につなげる
- HTML タグは `<p>`, `<h2>`, `<h3>`, `<hr>` のみ使用（リスト等は使わずに段落で書く。既存記事の文体に合わせる）

タイトル・slug・tags・metaDescription も生成：

- **タイトル**: `メインキーワード——補足説明` 形式（全角ダッシュ「——」で区切る）
- **slug**: kebab-case の英数字、25字以内目安
- **tags**: カンマ区切り、5個前後の日本語キーワード
- **metaDescription**: 80〜120字。疑問文・問いかけで始めることが多い

### 3. microCMS へ下書き投稿

`mcp__microcms__microcms_create_content_draft` で endpoint=`articles` に投稿する。
**サムネイルフィールドは送らない**（スキーマで任意に変更済み・サムネは管理画面で手動差し替え）。

カテゴリID は以下を使う：

| カテゴリ | ID |
|---|---|
| AI活用 | `832tdrrb1nu` |
| DX入門 | `6zm4okhu6y8a` |
| 業務効率化 | `1yfm3jzpp` |

投稿成功したら返ってきた contentId を控えておく。

### 4. ネタリスト更新

`knowledge/article-ideas.md` の該当行の `[ ]` を `[x]` に書き換える（Edit ツールで old_string をその行ピンポイントに）。

### 5. コミット & push

以下をまとめて1コミットにする：

- `knowledge/article-ideas.md`（チェック更新）

コミットメッセージ例:

```
feature: 記事「{タイトル}」を下書き投稿
```

その後 `git push` でリモートに反映する。

### 6. 結果報告

ユーザーに以下を報告して終わる：

- タイトル / カテゴリ / slug
- microCMS contentId
- 「管理画面でサムネ差し替え→公開してください」の一言

---

## 守るべきこと

- **被り厳禁**: microCMS の既存記事タイトルと意味的に重複していないか必ずチェック。slug も既存と被らないこと
- **CTA は自然に**: 「30分の無料相談」誘導は最後の段落のみ。本文中で何度も繰り返さない
- **凡庸な表現を避ける**: 「〜と言えるでしょう」「〜が大切です」みたいなフワッとした締めは禁止。具体的な数字・固有名詞・実例を盛り込む
- **既存記事の文体に合わせる**: 「です・ます調」、淡々とした業務メディアトーン、煽らない
